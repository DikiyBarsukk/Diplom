# Инструкция по развертыванию BARSUKSIEM

Для подробной схемы размещения файлов по ролям узлов, минимального состава каталогов и регламента эксплуатации используйте также РАЗМЕЩЕНИЕ_ФАЙЛОВ_И_ЭКСПЛУАТАЦИЯ.md.

## 1. Что разворачивается

Система состоит из трех основных режимов запуска:

- `python main.py server` - сервер и web UI
- `python main.py agent` - агент сбора логов
- `python main.py client` - desktop-клиент аудитора

## 2. Подготовка окружения

Требования:

- Python 3.11+
- доступ к сети между агентом и сервером
- для Linux-агентов - доступ к system journal или лог-файлам
- для Windows-агентов - доступ к Windows Event Log или лог-файлам

## 3. Установка зависимостей

### Сервер

```bash
python -m pip install -U pip
python -m pip install -U -r requirements-server.txt
```

### Клиент и агент

```bash
python -m pip install -U pip
python -m pip install -U -r requirements-client.txt
```

## 4. Настройка сервера

Рекомендуемые переменные окружения:

```bash
set BARSUKSIEM_BOOTSTRAP_ADMIN_USERNAME=admin
set BARSUKSIEM_BOOTSTRAP_ADMIN_PASSWORD=Admin123!Test
set BARSUKSIEM_COOKIE_SECURE=false
set BARSUKSIEM_COOKIE_SAMESITE=lax
set BARSUKSIEM_SESSION_MAX_AGE=86400
set CORS_ALLOWED_ORIGINS=http://127.0.0.1:8080,http://localhost:8080
```

Для Linux/macOS используйте `export ...` вместо `set`.

Если `BARSUKSIEM_BOOTSTRAP_ADMIN_PASSWORD` не задан, сервер создаст bootstrap admin автоматически и напечатает пароль в startup log.

## 5. Запуск сервера

```bash
python main.py server --host 0.0.0.0 --port 8080
```

Дополнительно поддерживается HTTPS:

```bash
python main.py server --host 0.0.0.0 --port 8443 --ssl-cert cert.pem --ssl-key key.pem
```

Проверка:

```bash
curl http://127.0.0.1:8080/health
```

## 6. Вход в web UI

1. Откройте `http://127.0.0.1:8080/login`
2. Войдите под bootstrap admin
3. После входа сервер выдаст session cookie
4. Все защищенные операции из UI будут выполняться с `X-CSRF-Token`

Основные страницы:

- `/`
- `/logs`
- `/incidents`
- `/analytics`
- `/inventory`
- `/compliance`

## 7. Запуск агента

### Автоматический выбор источника

```bash
python main.py agent --server http://127.0.0.1:8080 --source auto --interval 60
```

### Linux journal

```bash
python main.py agent --server http://127.0.0.1:8080 --source journal --interval 60
```

### Файлы логов

```bash
python main.py agent --server http://127.0.0.1:8080 --source file --interval 60
```

### UDP/TCP режим

```bash
python main.py agent --server udp://127.0.0.1:9000 --source auto
python main.py agent --server tcp://127.0.0.1:9001 --source auto
```

В HTTP режиме агент отправляет события на `POST /api/logs`.

## 8. Запуск desktop-клиента

Desktop-клиент работает с защищенными API-маршрутами и должен иметь учетные данные через env:

```bash
set BARSUKSIEM_CLIENT_USERNAME=admin
set BARSUKSIEM_CLIENT_PASSWORD=Admin123!Test
python main.py client --server http://127.0.0.1:8080
```

Клиент выполняет login через `/api/auth/login`, затем использует `/api/logs` и `/api/stats`.

## 9. Smoke-сценарий

1. Поднять сервер
2. Войти в `/login`
3. Запустить агента
4. Убедиться, что новые события появились в `/logs`
5. Убедиться, что инциденты видны в `/incidents`
6. Проверить сводную статистику на `/` и `/analytics`
7. Запустить тесты

## 10. Тесты

```bash
python -m unittest discover -s tests -v
```

Покрываются сценарии:

- login / me / logout
- ingest логов
- доступ к `/api/logs`, `/api/stats`, `/api/incidents`
- parser normalization
- incident rules

## 11. Типовые проблемы

### Не удается войти в систему

- проверьте startup log сервера и bootstrap credentials
- убедитесь, что база новая или учетная запись уже существует
- убедитесь, что cookie не блокируются браузером

### Агент не отправляет логи

- проверьте доступность `http://server:port/health`
- убедитесь, что агент использует корректный `--server`
- для HTTP режима используйте именно `http://...`, а не HTML route

### Desktop-клиент не работает

- задайте `BARSUKSIEM_CLIENT_USERNAME` и `BARSUKSIEM_CLIENT_PASSWORD`
- убедитесь, что клиент подключается к серверу, а не к HTML-странице

### CSRF ошибки

- logout и другие изменяющие операции должны идти через session cookie и `X-CSRF-Token`
- используйте `/api/auth/me` для получения актуального токена
