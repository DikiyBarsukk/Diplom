# 📋 Подробная инструкция по запуску приложения

## 🔍 Обзор проекта

Это клиент-серверное приложение для аудита логов с **распределенной архитектурой агентов**:

- **Агент** (agent) - собирает логи локально на любой машине (Windows/Linux) и отправляет на сервер
- **Сервер** - принимает логи от агентов, обрабатывает, нормализует и сохраняет в базу данных SQLite
- **Клиент-аудитор** (client) - GUI приложение для просмотра всех логов с сервера

**Преимущества новой архитектуры:**
- ✅ Мониторинг множества машин с одного сервера
- ✅ Поддержка Windows и Linux
- ✅ Централизованное хранение и анализ
- ✅ История логов в базе данных

```bash
python --version
```

## 🖥️ Запуск СЕРВЕРА (центральная обработка логов)

Сервер может работать на любой машине (Linux/Windows/macOS), но рекомендуется Linux.

### Установка зависимостей

**На Linux:**
```bash
python3 -m pip install -U pip
python3 -m pip install -U -r requirements-server.txt
```

**На Windows:**
```powershell
python -m pip install -U pip
python -m pip install -U -r requirements-server.txt
```

### 🚀 Запуск сервера

```bash
python main.py server --host 0.0.0.0 --port 8080
```

**Параметры:**
- `--host 0.0.0.0` - сервер будет доступен со всех сетевых интерфейсов (можно использовать `127.0.0.1` для локального доступа)
- `--port 8080` - порт, на котором будет работать сервер (можно изменить)
- `--db logs.db` - путь к базе данных SQLite (по умолчанию `logs.db`)

**Пример вывода:**
```
INFO:     Started server process [12345]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8080 (Press CTRL+C to quit)
```

**Важно:** Сервер создаст базу данных `logs.db` автоматически при первом запуске.

## 🤖 Запуск АГЕНТА (сбор логов)

Агент собирает логи локально и отправляет их на сервер. Можно запустить на нескольких машинах одновременно.

### Установка зависимостей

**На Linux:**
```bash
python3 -m pip install -U pip
python3 -m pip install -U -r requirements-client.txt
```

**На Windows:**
```powershell
python -m pip install -U pip
python -m pip install -U -r requirements-client.txt

# Опционально (для лучшей производительности Windows Event Log):
pip install pywin32
```

### 🚀 Запуск агента

**На Linux:**
```bash
python main.py agent --server http://<IP-сервера>:8080 --source journal --interval 60
```

**На Windows:**
```powershell
python main.py agent --server http://<IP-сервера>:8080 --source eventlog --interval 60
```

**Параметры:**
- `--server` - URL сервера (например, `http://192.168.1.100:8080`)
- `--source` - источник логов:
  - `auto` - автоматическое определение (рекомендуется)
  - `journal` - Linux journalctl
  - `eventlog` - Windows Event Log
  - `file` - файлы логов (Linux)
- `--limit 200` - количество событий за раз (по умолчанию 200)
- `--interval 60` - интервал отправки в секундах (по умолчанию 60)

**Пример вывода:**
```
Agent started. Collecting logs from: eventlog
Sending to: http://192.168.1.100:8080
Interval: 60 seconds
Press Ctrl+C to stop

[2024-01-15 10:30:00] Sent 45 logs, saved 45, skipped 0
[2024-01-15 10:31:00] Sent 12 logs, saved 12, skipped 0
```

### ⚠️ Важные замечания для агентов:

**Linux:**
- Для чтения `journalctl` может потребоваться членство в группе `systemd-journal`
- Добавление пользователя в группу: `sudo usermod -aG systemd-journal $USER` (требуется перелогин)

**Windows:**
- Если `pywin32` не установлен, агент использует PowerShell (работает, но медленнее)
- Для некоторых событий может потребоваться права администратора

```powershell

python -m pip install -U -r requirements-client.txt
```

**На Linux/macOS:**
```bash
python3 -m pip install -U pip
python3 -m pip install -U -r requirements-client.txt
```

### 🚀 Запуск клиента

**На Windows:**

**На Linux/macOS:**

- Фильтрация по хосту
- Фильтрация по уровню важности (severity)
- Просмотр статистики
- Отображение логов в таблице

## 🔧 Решение проблем

### Проблема: "python: command not found"

**Решение:**
- Используйте `python3` вместо `python`
- Убедитесь, что Python установлен и добавлен в PATH

### Проблема: "Permission denied" при запуске скрипта

**Решение для Linux:**
```bash
chmod +x scripts/install_server.sh
```

**Решение для Windows:**
- Убедитесь, что PowerShell имеет права на выполнение скриптов
- Попробуйте запустить PowerShell от имени администратора

### Проблема: "ModuleNotFoundError"

**Решение:**
- Убедитесь, что вы установили зависимости: `pip install -r requirements-server.txt` (для сервера) или `pip install -r requirements-client.txt` (для клиента/агента)
- Проверьте, что используете правильную версию Python (3.10+)

### Проблема: Агент не может подключиться к серверу

**Решение:**
1. Убедитесь, что сервер запущен
2. Проверьте, что IP-адрес и порт указаны правильно
3. Проверьте файрвол - порт 8080 должен быть открыт
4. Убедитесь, что сервер запущен с `--host 0.0.0.0`, а не `127.0.0.1`

### Проблема: "Access denied" при чтении логов на Linux

**Решение:**
```bash
# Добавьте пользователя в группу systemd-journal
sudo usermod -aG systemd-journal $USER
# Перезайдите в систему
```

### Проблема: Windows Event Log не работает

**Решение:**
1. Установите `pywin32`: `pip install pywin32`
2. Если не помогает, попробуйте запустить от имени администратора
3. Проверьте, что PowerShell доступен

### Проблема: Клиент не видит логи

**Решение:**
1. Убедитесь, что агенты запущены и отправляют логи
2. Проверьте, что сервер работает и принимает логи
3. Попробуйте нажать "Stats" в GUI, чтобы увидеть статистику
4. Проверьте фильтры в GUI (может быть установлен фильтр по хосту/severity)

## 🎯 API эндпоинты сервера

Для разработчиков:

- `GET /health` - проверка доступности сервера
- `POST /logs` - отправка логов от агентов (используется агентами)
- `GET /logs?host=...&severity=...&since=...&limit=...&offset=...` - получение логов с фильтрацией (используется клиентом)
- `GET /stats` - статистика по логам

Примеры:

```bash
# Проверка здоровья
curl http://localhost:8080/health

# Получение статистики
curl http://localhost:8080/stats

# Получение логов
curl "http://localhost:8080/logs?limit=10&severity=err"
```
