# 🔒 Правила выявления инцидентов информационной безопасности

## Обзор

Система использует rule-based анализ для автоматического выявления инцидентов ИБ. Каждое правило содержит формулу, псевдокод и описание логики обнаружения.

**Версия:** 0.4  
**Модуль:** `server/incidents.py`

## Правило R001: Обнаружение Brute Force атак

### Описание
Выявляет множественные неудачные попытки входа в систему, что является признаком атаки подбора паролей.

### Формула
```
COUNT(failed_login_events) >= 5 
AND 
TIME_SPAN(events) <= 10 минут
→ INCIDENT(brute_force, HIGH)
```

### Псевдокод
```python
def detect_brute_force(events):
    failed_logins = []
    
    # Фильтруем неудачные попытки входа
    FOR EACH event IN events:
        IF event.message CONTAINS ("login_failed" OR "authentication failed"):
            failed_logins.append(event)
    
    # Группируем по хосту
    GROUP failed_logins BY host
    
    # Проверяем каждую группу
    FOR EACH host_group:
        IF COUNT(host_group) >= 5:
            # Проверяем временное окно
            first_time = MIN(host_group.timestamp)
            last_time = MAX(host_group.timestamp)
            
            IF (last_time - first_time) <= 10 минут:
                RETURN {
                    type: "brute_force",
                    severity: "HIGH",
                    host: host,
                    event_count: COUNT(host_group),
                    time_window: "10 minutes"
                }
    
    RETURN None
```

### Параметры
- **Порог:** 5 неудачных попыток
- **Временное окно:** 10 минут
- **Критичность:** HIGH

### Примеры событий
```
[2024-01-15 10:00:00] login_failed: user=admin, ip=192.168.1.50
[2024-01-15 10:00:15] login_failed: user=admin, ip=192.168.1.50
[2024-01-15 10:00:30] login_failed: user=admin, ip=192.168.1.50
[2024-01-15 10:00:45] login_failed: user=admin, ip=192.168.1.50
[2024-01-15 10:01:00] login_failed: user=admin, ip=192.168.1.50
→ ИНЦИДЕНТ: Brute Force Attack
```

## Правило R002: Несанкционированный доступ - вход администратора ночью

### Описание
Выявляет входы администратора в нерабочее время (00:00-06:00), что может указывать на компрометацию учетной записи.

### Формула
```
event.action == "login_success" 
AND 
event.role == "admin" 
AND 
EXTRACT_HOUR(event.timestamp) IN [0, 6)
→ INCIDENT(unauthorized_access, MEDIUM)
```

### Псевдокод
```python
def detect_night_admin_login(events):
    FOR EACH event IN events:
        IF event.action == "login_success":
            IF event.role == "admin":
                hour = EXTRACT_HOUR(event.timestamp)
                
                # Нерабочее время: 00:00 - 06:00
                IF hour >= 0 AND hour < 6:
                    RETURN {
                        type: "unauthorized_access",
                        severity: "MEDIUM",
                        title: "Suspicious Admin Login at Night",
                        timestamp: event.timestamp,
                        hour: hour
                    }
    
    RETURN None
```

### Параметры
- **Временной диапазон:** 00:00 - 06:00
- **Роль:** admin
- **Критичность:** MEDIUM

### Примеры событий
```
[2024-01-15 02:30:00] login_success: user=admin, role=admin, ip=192.168.1.100
→ ИНЦИДЕНТ: Suspicious Admin Login at Night
```

## Правило R003: Подозрительная активность - PowerShell с загрузкой

### Описание
Выявляет запуск PowerShell с одновременной загрузкой данных из сети, что является типичным признаком вредоносной активности.

### Формула
```
EXISTS(powershell_execution) 
AND 
EXISTS(network_download) 
AND 
TIME_DIFF(powershell_event, download_event) <= 5 минут
→ INCIDENT(suspicious_activity, HIGH)
```

### Псевдокод
```python
def detect_suspicious_powershell(events):
    powershell_events = []
    download_events = []
    
    # Фильтруем события PowerShell
    FOR EACH event IN events:
        IF event.process == "powershell" OR event.message CONTAINS "powershell":
            powershell_events.append(event)
        
        # Фильтруем события загрузки
        IF event.message CONTAINS ("download" OR "wget" OR "curl" OR 
                                    "invoke-webrequest" OR "downloadstring"):
            download_events.append(event)
    
    # Проверяем временную корреляцию
    correlation_window = 5 минут
    
    FOR EACH ps_event IN powershell_events:
        FOR EACH dl_event IN download_events:
            time_diff = ABS(ps_event.timestamp - dl_event.timestamp)
            
            IF time_diff <= correlation_window:
                RETURN {
                    type: "suspicious_activity",
                    severity: "HIGH",
                    title: "Suspicious PowerShell Activity with Download",
                    powershell_event: ps_event,
                    download_event: dl_event,
                    time_diff: time_diff
                }
    
    RETURN None
```

### Параметры
- **Процесс:** powershell, pwsh
- **Ключевые слова загрузки:** download, wget, curl, invoke-webrequest, downloadstring, downloadfile
- **Временное окно корреляции:** 5 минут
- **Критичность:** HIGH

### Примеры событий
```
[2024-01-15 10:00:00] Process started: powershell.exe, user=admin
[2024-01-15 10:00:30] Network activity: download from http://malicious.com/payload.ps1
→ ИНЦИДЕНТ: Suspicious PowerShell Activity with Download
```

## Правило R004: Обнаружение попыток удаления логов

### Описание
Выявляет попытки удаления или изменения логов, что является признаком попытки сокрытия следов атаки.

### Формула
```
event.message CONTAINS ("log deletion" OR "log modification" OR 
                        "journalctl --vacuum" OR "wevtutil cl")
→ INCIDENT(log_tampering, CRITICAL)
```

### Псевдокод
```python
def detect_log_tampering(events):
    tampering_keywords = [
        "log deletion", "удаление логов",
        "journalctl --vacuum", "wevtutil cl",
        "clear event log", "log modification",
        "rm /var/log", "del *.log", "clear-log"
    ]
    
    FOR EACH event IN events:
        message = LOWER(event.message)
        process = LOWER(event.process)
        
        FOR EACH keyword IN tampering_keywords:
            IF keyword IN message OR keyword IN process:
                RETURN {
                    type: "log_tampering",
                    severity: "CRITICAL",
                    title: "Log Tampering Attempt Detected",
                    event: event,
                    keyword: keyword
                }
    
    RETURN None
```

### Параметры
- **Ключевые слова:** log deletion, journalctl --vacuum, wevtutil cl, clear event log, rm /var/log, del *.log
- **Критичность:** CRITICAL

### Примеры событий
```
[2024-01-15 10:00:00] Command executed: journalctl --vacuum-time=1d
→ ИНЦИДЕНТ: Log Tampering Attempt Detected

[2024-01-15 10:00:00] Command executed: wevtutil cl Security
→ ИНЦИДЕНТ: Log Tampering Attempt Detected
```

## Правило R005: Обнаружение повышения привилегий (корреляция событий)

### Описание
Выявляет последовательность событий, указывающую на повышение привилегий после подбора пароля.

### Формула (корреляция)
```
[4625] → [4625] → [4624] → [4672]
= 
failed_login → failed_login → successful_login → privileged_login
AND
TIME_SPAN(sequence) <= 30 минут
→ INCIDENT(privilege_escalation, CRITICAL)
```

### Псевдокод
```python
def detect_privilege_escalation(events):
    # Windows Event IDs:
    # 4625 = Failed login attempt
    # 4624 = Successful login
    # 4672 = Login with special privileges
    
    failed_logins = FILTER events WHERE event_id == 4625
    successful_logins = FILTER events WHERE event_id == 4624
    privileged_logins = FILTER events WHERE event_id == 4672
    
    correlation_window = 30 минут
    
    # Ищем последовательность
    FOR EACH failed1 IN failed_logins:
        FOR EACH failed2 IN failed_logins:
            IF failed2.timestamp > failed1.timestamp:
                FOR EACH success IN successful_logins:
                    IF success.timestamp > failed2.timestamp:
                        time_diff1 = success.timestamp - failed1.timestamp
                        
                        IF time_diff1 <= correlation_window:
                            FOR EACH priv IN privileged_logins:
                                IF priv.timestamp > success.timestamp:
                                    time_diff2 = priv.timestamp - success.timestamp
                                    
                                    IF time_diff2 <= correlation_window:
                                        RETURN {
                                            type: "privilege_escalation",
                                            severity: "CRITICAL",
                                            title: "Privilege Escalation Detected",
                                            pattern: "Failed Login → Failed Login → Successful Login → Privileged Login",
                                            events: [failed1, failed2, success, priv],
                                            total_time: priv.timestamp - failed1.timestamp
                                        }
    
    RETURN None
```

### Параметры
- **Паттерн:** 2 неудачных входа → 1 успешный вход → 1 вход с правами
- **Временное окно корреляции:** 30 минут
- **Критичность:** CRITICAL

### Примеры событий (Windows Event Log)
```
[2024-01-15 10:00:00] Event 4625: Failed login, user=admin, ip=192.168.1.50
[2024-01-15 10:00:15] Event 4625: Failed login, user=admin, ip=192.168.1.50
[2024-01-15 10:00:30] Event 4624: Successful login, user=admin, ip=192.168.1.50
[2024-01-15 10:01:00] Event 4672: Login with special privileges, user=admin
→ ИНЦИДЕНТ: Privilege Escalation Detected
```

## Уровни критичности

| Уровень | Описание | Время реакции |
|---|---|---|
| `CRITICAL` | Критический инцидент, требует немедленного реагирования | < 1 часа |
| `HIGH` | Высокий приоритет, требует быстрого реагирования | < 4 часов |
| `MEDIUM` | Средний приоритет, требует внимания | < 24 часов |
| `LOW` | Низкий приоритет, информационный | < 72 часов |
| `INFO` | Информационное сообщение | Мониторинг |

## Корреляция событий

Система поддерживает простую корреляцию событий по времени:

### Алгоритм корреляции

```python
def correlate_events(event_sequence, time_window):
    """
    Коррелирует последовательность событий в пределах временного окна.
    
    Args:
        event_sequence: Список событий в порядке появления
        time_window: Максимальное время между событиями
    
    Returns:
        True если события коррелированы
    """
    IF LENGTH(event_sequence) < 2:
        RETURN False
    
    first_time = event_sequence[0].timestamp
    last_time = event_sequence[-1].timestamp
    
    total_time = last_time - first_time
    
    IF total_time <= time_window:
        # Проверяем, что события идут последовательно
        FOR i IN RANGE(1, LENGTH(event_sequence)):
            time_diff = event_sequence[i].timestamp - event_sequence[i-1].timestamp
            IF time_diff < 0:  # События не в хронологическом порядке
                RETURN False
        
        RETURN True
    
    RETURN False
```

### Примеры корреляции

**Пример 1: Brute Force**
```
[10:00:00] Failed login
[10:00:15] Failed login
[10:00:30] Failed login
[10:00:45] Failed login
[10:01:00] Failed login
→ Корреляция: 5 событий за 1 минуту → Brute Force
```

**Пример 2: Privilege Escalation**
```
[10:00:00] Event 4625 (Failed login)
[10:00:15] Event 4625 (Failed login)
[10:00:30] Event 4624 (Successful login)
[10:01:00] Event 4672 (Privileged login)
→ Корреляция: последовательность за 1 минуту → Privilege Escalation
```

## Интеграция в систему

Анализ инцидентов выполняется автоматически при сохранении новых событий:

1. **Сбор событий** → Агенты собирают логи
2. **Нормализация** → События нормализуются
3. **Сохранение** → События сохраняются в БД
4. **Анализ** → Применяются правила выявления инцидентов
5. **Сохранение инцидентов** → Обнаруженные инциденты сохраняются в таблицу `incidents`
6. **Уведомления** → Инциденты логируются с уровнем WARNING

## Расширение системы

Для добавления новых правил:

1. Создайте класс, наследующий `IncidentRule`
2. Реализуйте метод `check()`
3. Добавьте правило в список `IncidentAnalyzer.rules`

**Пример:**
```python
class CustomRule(IncidentRule):
    def __init__(self):
        super().__init__(
            rule_id="R006",
            name="Custom Rule",
            description="Описание правила",
            severity=IncidentSeverity.MEDIUM,
            incident_type=IncidentType.ANOMALY
        )
    
    def check(self, events, context=None):
        # Логика проверки
        if condition:
            return {
                "rule_id": self.rule_id,
                "incident_type": self.incident_type,
                "severity": self.severity,
                "title": "Custom Incident",
                "description": self.description,
            }
        return None
```
