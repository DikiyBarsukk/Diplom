# 🐧 Рекомендации по выбору Linux-машины для аудита

## 📋 Требования системы

### Минимальные требования:
- **Python 3.10+** (обязательно)
- **systemd** (для journalctl - требуется для сбора логов)
- **SQLite3** (встроен в Python)
- **Сетевое подключение** (для связи с сервером)
- **512 MB RAM** (минимум для агента)
- **100 MB свободного места** (для зависимостей и логов)

### Рекомендуемые требования:
- **Python 3.11+**
- **1 GB RAM** (для комфортной работы)
- **500 MB свободного места**
- **Доступ к интернету** (для установки зависимостей)

```bash
python3 --version
```

**Проверка journalctl:**
```bash
journalctl --version
# Должен быть доступен
```

## 🎯 Рекомендуемые виртуальные машины

### 1. **VirtualBox** ⭐ **РЕКОМЕНДУЕТСЯ для начинающих**

**Преимущества:**
- ✅ Бесплатный и открытый исходный код
- ✅ Простая установка
- ✅ Хорошая производительность
- ✅ Поддержка всех ОС (Windows, Linux, macOS)
- ✅ Снимки состояния (snapshots)

**Системные требования:**
- 4 GB RAM (минимум для хоста)
- 50 GB свободного места
- Аппаратная виртуализация (VT-x/AMD-V)

**Рекомендуемые настройки для Ubuntu VM:**
- **RAM:** 2 GB (минимум 1 GB)
- **Диск:** 20 GB (динамический)
- **Процессоры:** 2 ядра
- **Сеть:** NAT или Bridged (для доступа к серверу)

**Установка:**
1. Скачайте [VirtualBox](https://www.virtualbox.org/)
2. Установите
3. Создайте новую VM с Ubuntu 22.04
4. Выделите 2 GB RAM, 20 GB диска
5. Установите Ubuntu

## 🔧 Настройка виртуальной машины

### Шаг 1: Создание VM

**VirtualBox:**
1. Создайте новую VM
2. Выберите тип: Linux, Ubuntu (64-bit)
3. RAM: 2048 MB
4. Диск: 20 GB (динамический)
5. Включите аппаратную виртуализацию

### Шаг 2: Установка Ubuntu

1. Скачайте Ubuntu 22.04 LTS ISO
2. Подключите ISO к VM
3. Запустите установку
4. Выберите минимальную установку (или серверную версию)

### Шаг 3: Настройка сети

**Для доступа к серверу из хоста:**
- Используйте **Bridged Adapter** или **NAT с пробросом портов**

**Bridged Adapter (рекомендуется):**
- VM получит IP в той же сети, что и хост
- Легко подключиться к серверу

**NAT с пробросом портов:**
- Нужно настроить проброс порта 8080
- Более сложно, но безопаснее

### Шаг 4: Установка зависимостей

```bash
# Обновление системы
sudo apt update && sudo apt upgrade -y

# Установка Python и pip
sudo apt install python3 python3-pip -y

# Проверка версии
python3 --version

# Добавление пользователя в группу systemd-journal
sudo usermod -aG systemd-journal $USER
newgrp systemd-journal

# Проверка journalctl
journalctl --version
```

---
### Для виртуальных машин:

1. **Производительность:** Виртуальные машины могут быть медленнее физических
2. **Сеть:** Убедитесь, что VM имеет доступ к сети для подключения к серверу
3. **Снимки:** Используйте snapshots для быстрого отката изменений
4. **Ресурсы:** Не перегружайте хост, выделяйте разумное количество RAM
