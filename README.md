# BARSUKSIEM

Система аудита системных журналов удаленного персонального компьютера для контроля информационной безопасности.

## Назначение

BARSUKSIEM собирает журналы с удаленных ПК, нормализует события, выявляет инциденты ИБ и предоставляет веб‑интерфейс для анализа, отчетности и комплаенса.

## Ключевые возможности

- Централизованный сбор логов с Linux/Windows
- Нормализация и хранение событий в SQLite
- Автоматическое выявление инцидентов ИБ
- Поиск и фильтрация по событиям и инцидентам
- Веб‑интерфейс с отчетами, инвентаризацией и комплаенсом
- Экспорт данных в CSV
- Аутентификация, CSRF‑защита, аудит действий

## Архитектура

- **Агенты**: сбор логов на удаленных ПК
- **Сервер**: FastAPI, хранение в SQLite, анализ инцидентов
- **Веб‑интерфейс**: HTML/JS + Tailwind через CDN

## Быстрый старт

### Сервер

1. Установите зависимости:

```
python -m pip install -U pip
python -m pip install -U -r requirements-server.txt
```

2. Запустите сервер:

```
python main.py server --host 0.0.0.0 --port 8080
```

3. Откройте веб‑интерфейс:

```
http://localhost:8080
```

### Агент

1. Установите зависимости:

```
python -m pip install -U pip
python -m pip install -U -r requirements-client.txt
```

2. Запуск агента:

```
python main.py agent --server http://<IP-СЕРВЕРА>:8080 --source journal --interval 60
```

## Веб‑разделы

- `/` — Главная страница аудита ИБ
- `/logs` — Журналы событий
- `/incidents` — Управление инцидентами
- `/incidents/details?id=...` — Карточка инцидента
- `/analytics` — Отчеты и аналитика
- `/inventory` — Удаленные ПК
- `/compliance` — Отчеты аудита и соответствие

## Основные API эндпоинты

- `POST /logs` — прием логов от агентов (публичный)
- `GET /api/logs` — события с фильтрацией
- `GET /stats` — статистика по логам
- `GET /api/incidents` — инциденты с фильтрацией/поиском
- `GET /api/incidents/stats` — статистика по инцидентам
- `GET /api/agents/stats` — онлайн/оффлайн агентов

Полная спецификация: `API_ДОКУМЕНТАЦИЯ.md`.
