## Краткий состав файлов по ролям

### Серверная ВМ

Нужны:

- `main.py`
- `requirements-server.txt`
- `server/`
- `common/`
- `web/`

Не нужны:

- `tests/`
- `client/`
- `.venv/`
- `.idea/`

### Агентная ВМ

Нужны:

- `main.py`
- `requirements-client.txt`
- `client/agent.py`
- `client/encryption.py`
- `client/udp_client.py`
- `common/`

Не нужны:

- `server/`
- `web/`
- `tests/`
- `client/gui.py`
- `client/connection.py`

### ВМ аудитора

Нужны:

- `main.py`
- `requirements-client.txt`
- `client/gui.py`
- `client/connection.py`
- `client/encryption.py`
- `client/udp_client.py`

Не нужны:

- `server/`
- `web/`
- `tests/`
- `common/`

## Краткий порядок запуска

1. На серверной ВМ поднять `python main.py server --host 0.0.0.0 --port 8080`.
2. На ВМ аудитора войти в `http://SERVER_IP:8080/login`.
3. На агентной ВМ запустить `python main.py agent --server http://SERVER_IP:8080 --source auto --interval 60`.
4. Проверить `/logs` и `/incidents`.
5. При необходимости на ВМ аудитора запустить `python main.py client --server http://SERVER_IP:8080`.

## Эксплуатационные напоминания

- База SQLite должна лежать только на серверной ВМ.
- `web/` нужен только серверу.
- Для первого запуска используйте HTTP и порт `8080`.
- Для агента используйте `--source auto`, если нет особой причины выбирать другой режим.
- Перед обновлением сервера делайте резервную копию файла базы.
